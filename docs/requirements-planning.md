## 1. 项目概述

### 1.1 产品定位

DailyRecord 是一款面向 macOS 平台的工作痕迹记录工具。核心价值在于：通过**定时提醒 + 轻量记录**的机制，帮助工作者持续追踪自己一天中的工作事项、进度与状态，从而形成可回溯、可分析的工作时间线。

### 1.2 核心解决痛点

| 痛点 | 解决方案 |
|------|----------|
| 工作中容易忘记自己做了什么 | 每 20 分钟弹窗提醒记录当前工作 |
| 日报/周报难写 | 所有记录自动汇聚，形成可查阅的时间线 |
| 多任务切换时进度混乱 | 统一管理所有工作事项的状态和进度 |
| 自制力不足，容易被带偏 | 定时提醒机制形成节律感，强化时间感知 |

### 1.3 设计目标

- **低摩擦**：记录流程需尽量快捷，单次操作不超过 10 秒
- **高可回溯**：所有数据持久化存储，支持查阅历史
- **美观易用**：UI 设计参考 TickTick 等成熟产品，达到审美及格线以上
- **本地优先**：数据存储在本地文件，不依赖外部服务，保护隐私

## 2. 功能需求

### 2.1 核心功能

#### F1: 定时弹窗提醒

| 项目 | 说明 |
|------|------|
| 触发间隔 | 每 20 分钟（可配置） |
| 提醒形式 | macOS 原生右上角通知弹窗 |
| 弹窗内容 | 提示文案：「时间到啦，记录一下当前在做什么？」 |
| 弹窗按钮 | **「填写」** 和 **「本次跳过」** |

**行为流程：**

```
触发时间到
    → 弹出 macOS 右上角通知
        → 点击「本次跳过」→ 关闭通知，等待下一轮
        → 点击「填写」→ 窗口扩展为居中记录窗口
            → 填写事项 + 进度 → 保存 → 关闭
```

> 若跳过，该轮次标记为「已跳过」，可在后续补齐。

#### F2: 事项记录窗口

| 项目 | 说明 |
|------|------|
| 触发方式 | 点击弹窗中的「填写」 |
| 窗口行为 | 从通知区扩展 → 变为居中的完整填写窗口 |
| 窗口大小 | 约 600×500 px（自适应） |

**表单字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 当前事项 | 下拉选择 + 可新增 | 下拉列表展示已有历史事项，支持键入新事项 |
| 事项分类 | 标签/分类选择 | 如「开发」「会议」「文档」「其他」 |
| 完成进度 | 手动输入 + 快捷选项 | 快捷选项如：25%、50%、75%、90% |
| 备注/进展 | 多行文本 | 描述当前进展、遇到的问题等 |
| 紧急程度 | 选择（高/中/低） | 可选，标记事项优先级 |

#### F3: 管理工作台

类似 TickTick 的管理界面，功能包括：

| 功能模块 | 说明 |
|----------|------|
| **看板视图** | 按「待办 / 处理中 / 已完成」三列展示事项卡片 |
| **事项列表** | 以列表方式展示所有工作事项 |
| **事项详情** | 点击事项查看/编辑详情（包含所有历史记录） |
| **进度管理** | 为每个事项设置和更新进度百分比 |
| **优先级标记** | 高/中/低 三级优先级，视觉区分 |
| **历史时间线** | 按时间顺序展示一天中的所有记录条目 |

**事项属性模型：**

```text
- 事项ID（唯一）
- 事项名称
- 分类标签
- 优先级（高/中/低）
- 状态（待办 / 处理中 / 已完成 / 已取消）
- 进度百分比（0% ~ 100%）
- 创建时间
- 更新时间
- 关联记录列表（每次弹窗填写的内容）
```

> 参考对象：TickTick 的任务管理 + 专注/记录功能。

### 2.2 辅助功能

#### F4: 数据持久化

- 数据存储于运行系统本地的文件中
- 存储格式：JSON 或 SQLite
- 支持导出备份
- 数据目录：`~/.dailyrecord/` 或程序同目录下的 `data/` 文件夹

#### F5: 配置管理

配置文件位于程序运行目录下，支持以下配置项：

```yaml
# config.yaml 示例
reminder:
  interval_minutes: 20           # 提醒间隔（分钟）
  enabled: true                  # 是否启用提醒
  skip_once: false               # 是否仅跳过下一次

storage:
  path: "./data"                 # 数据存储路径
  format: "json"                 # 存储格式 (json/sqlite)

appearance:
  theme: "auto"                  # 主题 (light/dark/auto)
  language: "zh-CN"              # 语言

notification:
  sound: true                    # 提醒是否播放声音
```

#### F6: 数据统计与分析（远期规划）

- 日/周/月工时统计
- 各类事项耗时占比
- 专注度分析（跳过率 vs 填写率）

## 3. 技术方案

### 3.1 总体架构

```
┌─────────────────────────────────────────────────┐
│                   macOS DMG                     │
│                                                   │
│  ┌─────────────┐    ┌─────────────────────────┐  │
│  │  Electron    │    │  Python 后端             │  │
│  │  (前端窗口)   │◄──►│  (FastAPI / Flask)      │  │
│  │             │    │  - 定时调度 (APScheduler) │  │
│  │  - 管理界面  │    │  - 数据持久化            │  │
│  │  - 记录窗口  │    │  - 通知推送              │  │
│  │  - 配置页面  │    │  - API 服务              │  │
│  └─────────────┘    └──────────┬──────────────┘  │
│                                │                   │
│                     ┌──────────▼──────────┐       │
│                     │  本地文件存储         │       │
│                     │  (JSON / SQLite)     │       │
│                     └─────────────────────┘       │
└─────────────────────────────────────────────────┘
```

### 3.2 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 后端 | **Python 3.10+** | 用户指定 |
| Web 框架 | **FastAPI** | 轻量、异步、自动 API 文档 |
| 定时调度 | **APScheduler** | 成熟的 Python 定时任务库 |
| 前端 | **React + TypeScript** | 生态丰富，UI 组件多 |
| 前端 UI 框架 | **Ant Design 或 Shadcn/ui** | 开箱即用，美观度高 |
| 桌面壳 | **Electron** | 成熟的跨平台桌面方案，打包 DMG 方便 |
| 通知 | **Electron Notification API** | macOS 原生通知支持 |
| 数据存储 | **SQLite (via SQLAlchemy)** | 本地持久化，无需额外服务 |
| 打包 | **electron-builder** | 支持 DMG 打包 |

### 3.3 备选方案

如果 Electron 体积过大，可考虑备选：

| 方案 | 说明 |
|------|------|
| **PyQt6 / PySide6** | Python 原生桌面，无前端依赖，界面较传统 |
| **Tauri** | Rust 壳 + Web 前端，体积小但需 Rust 编译环境 |
| **Python + webview** | 通过 pywebview 嵌入 Web 页面，介于两者之间 |

> **推荐首选**：Electron + React + Python FastAPI，开发效率最高，UI 表现力最强，生态最成熟。

## 4. 交互流程设计

### 4.1 主流程

```
程序启动
    → 启动 Python 后端服务（FastAPI）
    → 启动 Electron 窗口（管理界面）
    → 启动 APScheduler 定时器（20 分钟间隔）
    
    → [定时触发] → 发送通知给 Electron
        → Electron 弹出 macOS 右上角通知
            → 用户点击「填写」→ 打开记录窗口
            → 用户点击「跳过」→ 关闭，等待下一轮
    
    → 用户打开管理界面
        → 查看所有事项
        → 管理待办/处理中/已完成事项
        → 查看历史时间线
    
    → 用户关闭窗口
        → 后端继续运行（菜单栏驻留）
        → 右键菜单栏图标 → 退出程序
```

## 5. 数据模型设计

### 5.1 事项表 (Task)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT (UUID) | 主键 |
| name | TEXT | 事项名称 |
| category | TEXT | 分类标签 |
| priority | INTEGER | 优先级 1-3 (高-中-低) |
| status | TEXT | todo / in_progress / done / cancelled |
| progress | INTEGER | 0-100 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |
| is_active | BOOLEAN | 是否激活（软删除用） |

### 5.2 记录表 (Record)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT (UUID) | 主键 |
| task_id | TEXT | 关联事项 ID |
| progress | INTEGER | 本次记录时的进度 |
| note | TEXT | 备注/进展描述 |
| status | TEXT | filled / skipped |
| created_at | DATETIME | 记录时间 |

### 5.3 配置 (Config)

以 YAML 或 JSON 文件存储，见 2.2 节 F5。

## 6. 非功能需求

| 需求 | 说明 |
|------|------|
| 性能 | 后端内存占用 < 100MB，前端响应 < 500ms |
| 启动速度 | 程序冷启动 < 3 秒 |
| 数据安全 | 数据存储于本地，无网络传输（除非用户主动导出） |
| 稳定性 | 定时器需稳定可靠，错过提醒应进行补偿 |
| 隐私 | 不上传任何数据，无埋点 |
| 打包 | 最终产物为 .dmg 安装文件，符合 macOS 签名要求 |

## 7. 里程碑规划

| 阶段 | 内容 | 预计工作量 |
|------|------|-----------|
| **P0 MVP** | Python 后端 + FastAPI + SQLite + 定时提醒 | 3-5 天 |
|  | Electron 壳 + 右上角通知 + 记录窗口 | 3-5 天 |
|  | 本地数据持久化 + 配置管理 | 1-2 天 |
| **P1** | 管理工作台（看板视图 + 时间线） | 3-4 天 |
|  | 事项 CRUD + 进度管理 | 2-3 天 |
| **P2** | UI 美化（主题、动效、交互优化） | 2-3 天 |
|  | DMG 打包 + 安装流程 | 1-2 天 |
| **P3（可选）** | 数据统计与分析 | 2-3 天 |
|  | 数据导出（CSV/JSON） | 1 天 |

### 7.1 推荐启动顺序

1. **先搭后端骨架**：FastAPI 服务 + 数据模型 + CRUD API
2. **再搭前端架子**：Electron + React + 管理界面基础布局
3. **实现通知流程**：APScheduler → Electron → Notification
4. **实现记录窗口**：从通知到填写窗口的完整流程
5. **逐步打磨**：UI 美化、配置、统计

## 8. 技术风险和应对

| 风险 | 应对 |
|------|------|
| Electron 打包体积过大 | 考虑 Tauri 或 pywebview 备选方案 |
| macOS 通知权限限制 | 使用 Electron Notification API，在初次启动时引导用户授权 |
| Python 进程守护（后台运行） | 使用 macOS LaunchAgent 或 electron 主进程管理 Python 子进程 |
| 打包 Python 运行时 | 使用 PyInstaller 打包 Python 为独立二进制，或用 embedded Python |
| 定时器漂移 | APScheduler 自带 misfire 处理机制 |
